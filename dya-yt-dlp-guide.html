1. Install Python: Ensure you have Python installed on your system.

You can either download it from python.org OR Run the following in Powershell (Win + X then choose Windows Powershell (Admin)): Invoke-WebRequest -Uri "https://www.python.org/ftp/python/3.12.4/python-3.12.4-amd64.exe" -OutFile "python-installer.exe"; Start-Process -FilePath "python-installer.exe" -ArgumentList "/quiet InstallAllUsers=1 PrependPath=1" -Wait; Remove-Item "python-installer.exe" (edited)
11:06 AM

2. Install ffmpeg and ffprobe: To handle various media formats, yt-dlp relies on ffmpeg and ffprobe.

Open Powershell as Admin (Win + X then choose Windows Powershell (Admin)) and then run these following commands one at a time: 1. Download the ffmpeg ZIP file: Invoke-WebRequest -Uri "https://www.gyan.dev/ffmpeg/builds/ffmpeg-release-essentials.zip" -OutFile "ffmpeg.zip" 2. Extract the ZIP file: Expand-Archive -Path "ffmpeg.zip" -DestinationPath ".\ffmpeg" 3. Move the binaries to a directory in your PATH: Move-Item -Path ".\ffmpeg\ffmpeg-*-essentials_build\bin\*" -Destination "C:\ffmpeg" -Force 4. Add the directory to your PATH environment variable: [System.Environment]::SetEnvironmentVariable("Path", $([System.Environment]::GetEnvironmentVariable("Path", "Machine") + ";C:\ffmpeg"), "Machine") 5. Cleanup: Remove-Item -Path "ffmpeg.zip" then Remove-Item -Path ".\ffmpeg" -Recurse Ensure each command completes successfully before moving to the next one. Close Powershell before moving on to the next step so that we make sure the new PATH variables takes effect. (edited)

3. Install pip: Python's package installer used to install yt-dlp later on.

Open Powershell as Admin (Win + X then choose Windows Powershell (Admin)) and then run these following commands one at a time: 1. Download get-pip.py: Invoke-WebRequest -Uri "https://bootstrap.pypa.io/get-pip.py" -OutFile "get-pip.py" 2. Run the script to install pip: python get-pip.py 3. Cleanup: Remove-Item -Path "get-pip.py" Ensure each command completes successfully before moving to the next one. Close Powershell before moving on to the next step so that we make sure the new PATH variables takes effect. (edited)

4. Install yt-dlp: Finally we get to install yt-dlp. There are 2 main ways you can go about this:

FIRST OPTION:
  • You can download the yt-dlp standalone x64 binary (a .exe file) using this link
  • Then choose somewhere to create a dedicated folder where you will put this yt-dlp.exe, for example create a folder on your desktop and call it youtube, then put the yt-dlp.exe inside that folder.
  • Open the youtube folder, click in the address bar at the top of your file explorer, clear it out, type cmd and press enter, a command window should pop up and show you the file path to your youtube folder like for example: C:\Users\username\Desktop\youtube>.
  • Verify yt-dlp by running the following in the command window that opened up:
yt-dlp --version You should see something like 2024.07.16 as output for your command you just ran : : : now you are basically ready to use yt-dlp, to see it in action type the following in your open command window: yt-dlp <video_url> (replace <video_url> with the url to your chosen video, it should look something like: yt-dlp https://www.youtube.com/watch?v=5aYwU4nj5QA) : : : after you have run the command and it finished, you should see the video downloaded in your youtube folder, well done! ### You can add the path to your youtube folder in your PATH system variables if you want, if you do that means you can run the yt-dlp command from any directory in your system. Go read the following: How To Add a Directory or Program to Windows PATH. (edited)

Updating yt-dlp:

1. Have a command window open in your youtube folder directory if you followed the FIRST OPTION, otherwise just open a command window. 2. If you followed the FIRST OPTION, run the following in your command window to update yt-dlp: yt-dlp -U 3. If you followed the SECOND OPTION, run the following in your command window to update yt-dlp: python -m pip install -U "yt-dlp[default]" 4. Wait for it to finish and it should have updated yt-dlp.

## Some basic usage of different Switches: (edited)
12:26 PM
1. Basic Command: yt-dlp <video_url> Replace <video_url> with the URL of the video you want to download. 2. Download Video with Best Quality: yt-dlp -f best <video_url> Note this Option/Switch can be a bit redundant as yt-dlp automatically downloads the best quality if you pass no -f switch. 3. Download Audio Only: yt-dlp -f bestaudio <video_url> 4. Specify Output Filename: yt-dlp -o "%(title)s.%(ext)s" <video_url> 5. Download Playlist: yt-dlp -i <playlist_url> -i continues downloading even if errors occur. 6. Download Subtitles: yt-dlp --write-subs --sub-lang en <video_url> 7. Merge Video and Audio: yt-dlp -f bestvideo+bestaudio --merge-output-format mp4 <video_url>

Some recommended yt-dlp Switches to use (espescially for archiving purposes):

# Lines starting with # are comments # Download only videos not listed in the archive file AND it records the IDs of all downloaded videos in it --download-archive "C:\Path\to\Archive\file.txt" # File containing URLs to download, one URL per line. Lines starting with "#", ";" or "]" are considered as comments and are ignored. `-i` continues downloading even if errors occur. --batch-file="C:\Path\to\list\of\links.txt" -i # The path where the files should be downloaded to (Root path of downloads) -P "C:\Path\to\directory\youtube" # Remux the video into another container --remux-video mkv # Write video metadata to a .info.json file, this is very important for **DYA Archive Contribution** as the bot works with these files to add the videos to the archive. (this file may contain personal information) --write-info-json # Write video description to a .description file --write-description # Write video annotations --write-annotations # Write thumbnail image to disk --write-thumbnail # Embed metadata to the video file. Also embeds chapters/infojson if present --add-metadata OR --embed-metadata # Write subtitle file --write-subs # Write playlist metadata in addition to the video metadata --write-playlist-metafiles # Gets all sub languages --all-subs # Embed subtitles in the video (only for mp4, webm and mkv videos) --embed-subs # Embed thumbnail in the video as cover art --embed-thumbnail # Number of seconds to sleep before each download. This is the minimum time to sleep when used along with --max-sleep-interval --min-sleep-interval 1 # Maximum number of seconds to sleep. Can only be used along with --min-sleep-interval --max-sleep-interval 15 # Minimum download rate in bytes per second below which throttling is assumed and the video data is re-extracted --throttled-rate 100K

CONFIGURATION

You can configure yt-dlp by placing any supported command line option to a configuration file. The configuration is loaded from the following locations: 1. Main Configuration:
  • The file given to --config-location
1. Portable Configuration: (Recommended for portable installations) < this is the FIRST OPTION
  • If using a binary, yt-dlp.conf in the same directory as the binary
1. Home Configuration:
  • yt-dlp.conf in the home path given to -P
  • If -P is not given, the current directory is searched
1:10 PM
E.g. with the following configuration file, yt-dlp will always extract the audio, not copy the mtime, use a proxy and save all videos under YouTube directory in your home directory: # Lines starting with # are comments # Always extract audio -x # Do not copy the mtime --no-mtime # Use this proxy --proxy 127.0.0.1:3128 # Save all videos under YouTube directory in your home directory -o ~/YouTube/%(title)s.%(ext)s
1:11 PM
Note: Options in configuration file are just the same options aka switches used in regular command line calls; thus there must be no whitespace after - or --, e.g. -o or --proxy but not - o or -- proxy. They must also be quoted when necessary, as if it were a UNIX shell. You can use --ignore-config if you want to disable all configuration files for a particular yt-dlp run. If --ignore-config is found inside any configuration file, no further configuration will be loaded. For example, having the option in the portable configuration file prevents loading of home, user, and system configurations. Additionally, (for backward compatibility) if --ignore-config is found inside the system configuration file, the user configuration is not loaded.
